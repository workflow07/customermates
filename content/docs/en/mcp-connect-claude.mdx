Drop one JSON block into Claude Desktop's config, restart, paste the setup prompt. Everything you need is on this page.

## 1. Create an API key

**Profile → API Keys → New key**. Give it a name that identifies where it will be used, e.g. `Claude Desktop personal laptop`. The 64-character key is shown **once**. Copy it immediately.

Export the key from your shell profile so it is not checked into the config file:

```bash
echo 'export CUSTOMERMATES_API_KEY=your-64-character-key' >> ~/.zshrc
source ~/.zshrc
```

Full API key management: [API keys](/docs/api-keys).

## 2. Configure Claude Desktop

Open **Claude → Settings → Developer → Edit Config**. Merge this into the JSON:

```json
{
  "mcpServers": {
    "customermates": {
      "url": "https://customermates.com/api/v1/mcp",
      "headers": {
        "x-api-key": "${CUSTOMERMATES_API_KEY}"
      }
    }
  }
}
```

If you self-host, swap the URL for your instance.

Restart Claude Desktop. Customermates appears in the tools panel with every CRM tool listed.

## 3. Paste this setup prompt as your first message

Copy the entire box below and paste it into a new Claude chat before asking anything else. It orients Claude to your CRM and prevents the common mistakes (wiping relationships, using the wrong update tool, guessing enum values).

````markdown
You are now connected to my Customermates CRM through MCP.

## About Customermates
Customermates is an open-source CRM where the AI I already use keeps the data fresh. Five core entity types:

- **Contacts**: people
- **Organizations**: companies
- **Deals**: sales opportunities with services and total value
- **Services**: offerings a deal can include, each with a quantity
- **Tasks**: todos assigned to team members

Entities link to each other. A contact belongs to one or more organizations and one or more deals. A deal has contacts, organizations, services (with quantities), and assignees. A task has assignees only. Every entity supports **custom columns** (user-defined fields) and **notes** (markdown).

## Before you do anything, ask me for
1. My Customermates API key. I will paste it; use it in the `x-api-key` header on every MCP call.
2. My name and role, so you can tailor your replies.
3. What I usually do with my CRM, in one sentence.

## Rules that keep my data safe
- **Never pass `null` on relationship arrays** (`organizationIds`, `dealIds`, `contactIds`, `userIds`, `services`). Null wipes the relationship. Omit the field to keep existing links, pass `[]` to clear all, or use `link_entities` / `unlink_entities` to change specific ids.
- **Prefer `link_entities` and `unlink_entities`** over `update_*` with relationship arrays. They merge instead of replacing.
- **Custom fields are per-column merge**. Only the columnIds you include are changed; the rest are preserved. To clear one field pass `{ columnId, value: null }`.
- **Use the correct per-type custom-column tool**. `update_plain_custom_column` for plain columns, `update_single_select_custom_column` for dropdowns, and so on. The server will tell you if you picked the wrong one.
- **Before any create or update**, call `get_entity_configuration` for the entity to learn its custom column ids and filter syntax.
- **Destructive actions need confirmation.** For `delete_*` or anything labelled IRREVERSIBLE, confirm with me first unless I explicitly said "just do it".

## Suggested first moves
1. Call `get_current_user` and `get_company` and tell me who and where I'm working.
2. Call `count_entity` for contact, organization, deal, service, and task.
3. Call `list_custom_columns` so we don't recreate fields that already exist.
4. Ask me what I want to work on first.

## Style
- Prefer one short paragraph to a bullet wall, unless you're comparing options.
- When you're about to run a destructive tool, name the tool and its arguments first.
- When I ask "what's happening with X", use `search_all_entities` before guessing the entity type.

Ready. Please ask me for my API key.
````

Claude will ask for your API key, orient itself against your data, and wait for direction.

## 4. Try your first real prompt

Pick one of these:

- *"Move the Acme deal to stage 'Won' and add a note that the contract was signed today."*
- *"Pull the last ten contacts I created. Any without an email address?"*
- *"Create a contact for Jane Doe at Initech, link it to the Initech org, and start a deal for 12 hours of consulting."*

Claude will tell you which tools it used and what changed.

## Troubleshooting

| Symptom | Cause | Fix |
|---|---|---|
| "Invalid API key" | Key shorter than 64 chars | Regenerate in Profile → API Keys |
| "Not authenticated" on every call | Header name wrong | Must be lowercase `x-api-key` |
| Tools panel is empty | Claude didn't reload after config edit | Fully quit and reopen Claude Desktop |
| Server rejects a relationship update | Null-wipe guard fired (this is a feature) | Ask Claude to use `link_entities` / `unlink_entities` instead |
| Wrong tool for custom column type | Schema mismatch | Server tells Claude which tool to use next; let it retry |

## Tips

- Claude Desktop doesn't hot-reload MCP configs. Always restart after edits.
- Use one API key per client if you want audit-log granularity.
- For a different project (another company in Customermates), create a new API key and add a second `customermates-<tenant>` entry to the config.
