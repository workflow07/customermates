## Self-host Customermates with Docker Compose

This page shows how to run a self-hosted Customermates deployment with Docker Compose and PostgreSQL.
If you are still deciding between deployment models, compare [Self-Hosted vs Cloud](/docs/self-host-vs-cloud) first.

Self-hosted deployments run the community edition. Enterprise features (Audit Logging, Single Sign-On, Whitelabeling) require a separate license.

## Prerequisites

- Docker Engine and Docker Compose on a VPS
- A domain pointed to your server
- A reverse proxy with HTTPS termination for production usage

## Installation

```bash
mkdir customermates && cd customermates
BASE=https://raw.githubusercontent.com/customermates/customermates/main
curl -O $BASE/docker-compose.yml
curl -O $BASE/.env.selfhost.template
curl -O $BASE/scripts/selfhost-{setup,update,restart,reset}.sh
chmod +x selfhost-*.sh
mv .env.selfhost.template .env
# edit .env with your values
./selfhost-setup.sh
```

The setup script validates your `.env`, pulls the app image, runs database migrations, and starts all services.

## Verify

```bash
docker compose ps
docker compose logs -f app
```
