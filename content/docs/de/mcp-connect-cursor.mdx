Cursor hat eine eingebaute MCP-Server-Auswahl. Zwei Klicks zum Hinzufügen, dann den Setup-Prompt einfügen.

## 1. API-Key anlegen

In Customermates: **Profil → API Keys → Neuer Key**. Gib ihm einen Namen (z.B. `Cursor`). Der 64-Zeichen-Key wird **einmal** angezeigt — sofort kopieren.

## 2. Server in Cursor hinzufügen

**Settings → Tools & MCP → Add new MCP server** öffnen. Folgendes einfügen, `YOUR_KEY` ersetzen:

<McpInstallSnippet tool="cursor" />

Bei Self-Host: URL durch deine Instanz ersetzen.

Cursor lädt hot — kein Neustart nötig. Die Customermates-Tools erscheinen direkt im Composer.

## Warum Customermates in Cursor

Der CRM-Stand ist Kontext. *"Zeig mir die Deals, die auf Engineering warten"* während du in einem Ticket bist, oder *"Log diesen Call"* während du das Repo des Kunden liest, hält dich im Flow. Der Agent liest oder schreibt, ohne dass du den Editor verlassen musst.

## Probier deinen ersten echten Prompt

- *"Verschieb den Acme-Deal in Stage 'Won' und füge eine Notiz hinzu, dass der Vertrag heute unterschrieben wurde."*
- *"Find Contacts in der Org für das Repo, in dem ich gerade bin."*
- *"Log einen Task, dem Bug-Report nachzugehen, den wir eben besprochen haben."*

## Manuelle Konfiguration (Alternative)

Wer die Datei direkt bearbeiten möchte: derselbe Block kommt in `~/.cursor/mcp.json` (global) oder `<project>/.cursor/mcp.json` (pro Projekt), eingebettet unter `mcpServers`:

```json
{
  "mcpServers": {
    "customermates": {
      "url": "https://customermates.com/api/v1/mcp",
      "headers": { "x-api-key": "YOUR_KEY" }
    }
  }
}
```

## Troubleshooting

| Symptom | Lösung |
|---|---|
| Server nicht aufgeführt | Settings → Tools & MCP öffnen, Eintrag prüfen, ggf. Window reloaden |
| "Invalid API key" | In Profil → API Keys neu erstellen |
| Tool-Calls laufen in Timeout | `HTTPS_PROXY` in der Shell setzen, in der Cursor gestartet wurde |
