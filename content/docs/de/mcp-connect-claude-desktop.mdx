Die "Custom Connectors"-UI in Claude Desktop unterstützt nur OAuth — keine statischen API-Header. Daher gehen wir über die Konfigurationsdatei mit dem `mcp-remote`-Shim. Ein JSON-Block, ein Neustart.

> Wenn du Claude Code (das CLI) nutzt, siehe die [Claude Code-Seite](/docs/mcp-connect-claude-code) — dort reicht ein einziger Terminal-Befehl.

## 1. API-Key anlegen

In Customermates: **Profil → API Keys → Neuer Key**. Gib ihm einen Namen, der den Client beschreibt (z.B. `Claude Desktop`). Der 64-Zeichen-Key wird **einmal** angezeigt — sofort kopieren.

## 2. Konfigurationsdatei bearbeiten

**Claude → Settings → Developer → Edit Config** öffnen, oder die Datei direkt:

- macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`
- Windows: `%APPDATA%\Claude\claude_desktop_config.json`

Diesen Block mergen, `YOUR_KEY` ersetzen:

<McpInstallSnippet tool="claudeDesktop" />

Bei Self-Host: URL durch deine Instanz ersetzen.

`mcp-remote` ist ein kleiner Shim, der Claude Desktop per stdio mit einem Remote-HTTP-MCP-Server sprechen lässt. Er wird beim ersten Aufruf via `npx` heruntergeladen; du brauchst Node 18+ im `PATH`.

## 3. Claude Desktop neu starten

Vollständig beenden (⌘Q auf macOS, nicht nur Fenster schließen) und neu öffnen. Customermates erscheint im Tools-Panel mit allen CRM-Tools.

## Probier deinen ersten echten Prompt

- *"Verschieb den Acme-Deal in Stage 'Won' und füge eine Notiz hinzu, dass der Vertrag heute unterschrieben wurde."*
- *"Hol die letzten zehn Contacts, die ich angelegt habe. Welche haben keine E-Mail-Adresse?"*
- *"Leg einen Contact für Jane Doe bei Initech an, verknüpf ihn mit der Initech-Org und starte einen Deal über 12 Stunden Beratung."*

## Troubleshooting

| Symptom | Ursache | Lösung |
|---|---|---|
| Tools-Panel ist leer | Claude wurde nicht neu geladen | Vollständig beenden und neu öffnen |
| `npx: command not found` | Kein Node im `PATH` | Node 18+ von [nodejs.org](https://nodejs.org) installieren |
| "Invalid API key" | Falsche Key-Länge | In Profil → API Keys neu erstellen |
| Server lehnt Relationship-Update ab | Null-Wipe-Schutz hat ausgelöst (gewollt) | Claude bitten, `link_entities` / `unlink_entities` zu nutzen |

## Tipps

- Claude Desktop lädt MCP-Configs nicht hot. Nach jeder Änderung beenden und neu öffnen.
- Pro Client einen eigenen API-Key, wenn du Audit-Log-Granularität willst.
- Für mehrere Customermates-Konten einen zweiten `customermates-<tenant>`-Block mit dem jeweiligen Key anlegen.
