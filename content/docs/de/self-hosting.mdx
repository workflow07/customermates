## Customermates self-hosted mit Docker Compose starten

Diese Seite zeigt, wie du eine self-hosted Customermates-Installation mit Docker Compose und PostgreSQL aufsetzt.
Wenn du noch zwischen Deployment-Modellen vergleichst, gehe zuerst zu [Self-Hosted vs Cloud](/docs/self-host-vs-cloud).

Self-hosted Deployments laufen in der Community Edition. Enterprise-Features (Audit Logging, Single Sign-On, Whitelabeling) erfordern eine separate Lizenz.

## Voraussetzungen

- Docker Engine und Docker Compose auf einem VPS
- Eine Domain, die auf deinen Server zeigt
- Ein Reverse Proxy mit HTTPS-Terminierung fuer produktive Nutzung

## Installation

```bash
mkdir customermates && cd customermates
BASE=https://raw.githubusercontent.com/customermates/customermates/main
curl -O $BASE/docker-compose.yml
curl -O $BASE/.env.selfhost.template
curl -O $BASE/scripts/selfhost-{setup,update,restart,reset}.sh
chmod +x selfhost-*.sh
mv .env.selfhost.template .env
# .env mit deinen Werten bearbeiten
./selfhost-setup.sh
```

Das Setup-Skript validiert deine `.env`, pullt das App-Image, fuehrt Datenbank-Migrationen aus und startet alle Services.

## Pruefen

```bash
docker compose ps
docker compose logs -f app
```
