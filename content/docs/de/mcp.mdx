MCP-Client auf `https://customermates.com/api/v1/mcp` zeigen, API-Key im Header `x-api-key`, und der Client entdeckt automatisch alle 57 CRM-Tools.

Wenn du das volle End-to-End-Setup für einen bestimmten Client willst (API-Key, JSON-Config, Setup-Prompt, erster Befehl, alles auf einer Seite), spring direkt zu [Claude](/docs/mcp-connect-claude-desktop), [ChatGPT](/docs/mcp-connect-chatgpt) oder [Cursor](/docs/mcp-connect-cursor). Diese Seite ist die Protokoll-Referenz.

## Wann MCP passt

- Deine KI soll das **CRM lesen und schreiben**, ohne dass du IDs kopierst.
- Die KI soll **Fähigkeiten entdecken**, statt dass du API-Calls von Hand schreibst.
- **Ein Endpoint** für Claude, ChatGPT, Cursor, Codex und jeden zukünftigen Client.

Nimm stattdessen [OpenAPI](/docs/openapi), wenn ein Entwickler oder Integrations-Service genau weiß, welchen Endpoint er braucht. OpenAPI ist die kanonische HTTP-Referenz. MCP ist die Agent-native Oberfläche darüber.

## Der Endpoint

```
POST https://customermates.com/api/v1/mcp
Content-Type: application/json
x-api-key: <dein-64-zeichen-key>
```

Der Endpoint spricht Model Context Protocol (Streamable-HTTP-Variante). `tools/list` liefert jedes Tool mit JSON Schema. `tools/call` ruft ein Tool per Name auf.

## Wie die Tool-Oberfläche für schwächere Modelle gebaut ist

Nicht jedes Modell plant so gut wie Claude Opus. Die MCP-Oberfläche ist auf den Weak-Model-Case zugeschnitten:

- **Verb-first-imperative Namen**: `create_contacts`, `update_deal`, `delete_custom_column`. Kein Batch-Präfix. Verb passt zur Absicht.
- **Schmale Per-Type-Tools**: acht `create_*_custom_column` und acht `update_*_custom_column` statt eines Discriminated-Union-Giganten. Modelle wählen per Name statt per Branch auf `type`.
- **Inline-Enum-Hinweise**: jedes Enum-Feld listet gültige Werte inline, kein Auflösen externer Typen.
- **Filter-Beispiele inline**: jeder `filters`-Parameter mit JSON-Beispiel in der Beschreibung.
- **Relationship-Sicherheit**: `link_entities` und `unlink_entities` mergen; `null` auf Relationship-Array wird mit Remediation-Hint abgelehnt.
- **Type-Mismatch-Routing**: falsches Per-Type-Update liefert "this column is type X, use `update_X_custom_column`".
- **Destructive Flags**: jedes `delete_*` hat `destructiveHint: true` und beginnt mit `IRREVERSIBLE`.

Die volle Tool-Liste: [MCP-Tool-Katalog](/docs/mcp-tool-catalog).

## CLI-Nutzung

Wenn du einen lokalen Client willst statt GUI, verbinden Tools wie `mcporter` denselben Endpoint. API-Key einmal in der Client-Config speichern und Tools aus der Shell aufrufen.

## OpenAPI neben MCP

Beide liegen an derselben Base-URL. MCP auf `/api/v1/mcp`; OpenAPI-Spec auf `/api/v1/openapi`. OpenAPI-Operationen mappen 1:1 auf REST-Endpoints; MCP-Tools wrappen diese plus Safety-Guardrails, die die Raw-API nicht hat.
