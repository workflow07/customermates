# n8n für Anfänger: Ersten Automatisierungs-Workflow bauen

n8n ist die Open-Source-Automatisierungsplattform, die Zapier und Make zu einem Bruchteil der Kosten ersetzt. Diese Anleitung führt Sie durch Installation, Editor-Navigation, ersten Workflow, Fehlerbehandlung und CRM-Integration.

## Was ist n8n?

[n8n](https://n8n.io/) (ausgesprochen "n-eight-n") ist eine Workflow-Automatisierungsplattform, die verschiedene Anwendungen und Dienste miteinander verbindet. Sie bauen Workflows visuell, indem Sie Nodes auf eine Arbeitsfläche ziehen und verbinden -- für die meisten Anwendungsfälle ist kein Code erforderlich.

Was n8n von anderen Automatisierungstools unterscheidet, ist das Deployment-Modell. Sie können n8n in der Cloud nutzen (gehostet von n8n) oder auf Ihrem eigenen Server selbst hosten. Self-Hosting bedeutet, dass Ihre Daten Ihre Infrastruktur nie verlassen -- relevant für DSGVO-Compliance und datensensible Branchen.

n8n bietet über 400 integrierte Schnittstellen (Nodes genannt), unterstützt benutzerdefinierten Code in JavaScript und Python und erlaubt die Anbindung beliebiger APIs über HTTP-Request-Nodes. Zusätzlich gibt es native Unterstützung für KI- und LLM-Nodes, was n8n zur natürlichen Wahl für Teams macht, die KI-gestützte Automatisierungen aufbauen wollen.

### Eckdaten zu n8n

- **Open Source** unter der Sustainable Use License (Quellcode auf [GitHub](https://github.com/n8n-io/n8n))
- **400+ Integrationen** einschließlich Google Workspace, Slack, Datenbanken, CRMs und KI-Modelle
- **Visueller Workflow-Editor** mit Drag-and-Drop-Arbeitsfläche
- **Self-Hosting** möglich via Docker, npm oder Kubernetes
- **Cloud-Option** für Teams, die Managed Hosting bevorzugen
- **KI-nativ** mit integrierten Nodes für OpenAI, Anthropic und lokale LLMs

## n8n Cloud vs. Self-Hosted: Preise und Abwägungen

Eine der ersten Entscheidungen ist, ob Sie n8n Cloud oder Self-Hosting nutzen. Beide Optionen haben klare Vorteile je nach Situation.

### n8n Cloud Preise (Stand 2026)

| Plan | Monatliche Kosten | Workflow-Ausführungen | Aktive Workflows | Wichtige Features |
|------|-------------------|----------------------|-----------------|-------------------|
| Starter | 20 EUR | 2.500 | 5 | Basis-Support, 1 Nutzer |
| Pro | 50 EUR | 10.000 | 15 | Priority-Support, 5 Nutzer, Versionshistorie |
| Enterprise | Individuell | Unbegrenzt | Unbegrenzt | SSO, Audit-Logs, dedizierter Support |

n8n Cloud ist der schnellste Weg zum Start. Kein Server-Setup, keine Wartung, automatische Updates. Innerhalb von Minuten nach der Anmeldung bauen Sie Workflows.

### Self-Hosted-Kosten

| Komponente | Monatliche Kosten |
|-----------|------------------|
| n8n-Software | Kostenlos (Open Source) |
| VPS (z. B. Hetzner, DigitalOcean) | 5-20 EUR |
| Domain + SSL (via Let's Encrypt) | Kostenlos |
| Wartungsaufwand | 1-2 Stunden/Monat |
| **Gesamt** | **5-20 EUR** |

Self-Hosting ist bei den Softwarekosten kostenlos, erfordert aber einen Server und grundlegendes DevOps-Wissen. Der Kompromiss: unbegrenzte Workflow-Ausführungen, volle Datenkontrolle und die Möglichkeit, interne Systeme anzubinden -- aber Sie übernehmen Updates, Backups und Monitoring selbst.

### Wann welche Option wählen

**Wählen Sie n8n Cloud, wenn:**
- Sie sofort loslegen wollen, ohne Server einzurichten
- Ihr Team keine DevOps-Erfahrung hat
- Sie weniger als 10.000 Ausführungen pro Monat benötigen
- Sie Managed Hosting mit automatischen Updates bevorzugen

**Wählen Sie Self-Hosting, wenn:**
- Daten auf Ihrer Infrastruktur bleiben müssen (DSGVO, Gesundheitswesen, Finanzen)
- Sie unbegrenzte Workflow-Ausführungen benötigen
- Sie interne Systeme hinter einer Firewall anbinden wollen
- Sie grundlegende Docker-/Server-Kenntnisse haben oder ein technisches Teammitglied

Die meisten Einsteiger beginnen mit n8n Cloud und migrieren zum Self-Hosting, wenn Nutzung und Erfahrung wachsen.

## n8n vs. Zapier vs. Make: Ein ehrlicher Vergleich

Alle drei Plattformen lösen dasselbe Grundproblem: Anwendungen verbinden, ohne Code zu schreiben. Aber sie unterscheiden sich erheblich in Preisgestaltung, Flexibilität und Datenkontrolle.

| Merkmal | n8n | Zapier | Make |
|---------|-----|--------|------|
| Preismodell | Self-Host kostenlos; Cloud ab 20 EUR/Monat | Ab 20 USD/Monat (begrenzte Tasks) | Ab 9 USD/Monat (begrenzte Operations) |
| Task-/Operations-Limits | Unbegrenzt bei Self-Hosting | Strenge Plan-Limits | Strenge Plan-Limits |
| Self-Hosting | Ja (Docker, npm) | Nein | Nein |
| Custom Code | JavaScript + Python | Eingeschränkt (Code by Zapier) | Eingeschränkt |
| KI/LLM-Nodes | Nativ (OpenAI, Anthropic, Ollama) | Über Plugins | Über Module |
| Datenstandort | Volle Kontrolle (Self-Host) | US-gehostet | EU-Option verfügbar |
| Open Source | Ja | Nein | Nein |
| Anzahl Integrationen | 400+ | 6.000+ | 1.500+ |

**Wählen Sie Zapier,** wenn Sie die breiteste Auswahl an vorgefertigten Integrationen brauchen und sich nicht mit Infrastruktur beschäftigen möchten. Rechnen Sie damit, dass die Kosten mit wachsendem Volumen steil ansteigen.

**Wählen Sie Make,** wenn Sie einen visuellen Builder mit feinerer Kontrolle als Zapier und moderater Preisgestaltung wollen. Makes Scenario-Editor ist leistungsfähig, hat aber eine steilere Lernkurve.

**Wählen Sie n8n,** wenn Sie volle Datenkontrolle, unbegrenzte Ausführungen beim Self-Hosting, KI/LLM-Integration oder schlicht Open Source bevorzugen. n8n ist die stärkste Wahl für technische Teams und für alle, die CRM-Automatisierung mit native KI-Integration verbinden möchten.

## n8n installieren: Drei Optionen

### Option 1: n8n Cloud (Schnellster Einstieg)

Gehen Sie zu [n8n.io](https://n8n.io/), erstellen Sie ein Konto, und Sie können sofort Workflows bauen. Keine Installation, keine Server-Verwaltung. Der Starter-Plan kostet 20 EUR/Monat und umfasst 2.500 Workflow-Ausführungen.

Dies ist die empfohlene Option für Einsteiger und für Teams, die n8n evaluieren möchten, bevor sie sich für Self-Hosting entscheiden.

### Option 2: Self-Hosting mit Docker (Empfohlen für Produktion)

Wenn Sie einen Server haben (ein VPS ab 5 EUR/Monat reicht), können Sie n8n mit einem einzigen Docker-Befehl starten:

```bash
docker run -it --rm \
  --name n8n \
  -p 5678:5678 \
  -v n8n_data:/home/node/.n8n \
  n8nio/n8n
```

Öffnen Sie dann `http://localhost:5678` im Browser. Für den Produktionsbetrieb fügen Sie einen Reverse Proxy (Nginx oder Caddy) hinzu, aktivieren HTTPS und richten persistenten Speicher ein. Die [n8n-Self-Hosting-Anleitung](https://docs.n8n.io/hosting/) beschreibt dies im Detail.

Self-Hosting bietet unbegrenzte Workflow-Ausführungen ohne Zusatzkosten, volle Datenkontrolle und die Möglichkeit, interne Systeme anzubinden, die nicht im Internet erreichbar sind.

### Option 3: Installation via npm

Für Entwickler, die n8n lokal ohne Docker betreiben möchten:

```bash
npm install n8n -g
n8n start
```

Dies installiert n8n global und startet den Server auf Port 5678. Geeignet für lokale Entwicklung und Tests, aber ohne zusätzliche Konfiguration nicht für den Produktionsbetrieb empfohlen.

## Die n8n-Editor-Oberfläche navigieren

Bevor Sie Ihren ersten Workflow bauen, nehmen Sie sich fünf Minuten Zeit, um die Editor-Oberfläche zu verstehen. Zu wissen, wo sich was befindet, spart Zeit und verhindert Verwirrung.

### Die Arbeitsfläche (Canvas)

Die Arbeitsfläche ist der Hauptarbeitsbereich, in dem Sie Workflows bauen. Es ist ein unbegrenzter Raum, in dem Sie Nodes ziehen, platzieren und verbinden. Zoomen Sie mit dem Scrollrad und verschieben Sie die Ansicht durch Klicken und Ziehen auf leeren Bereich.

Wichtige Interaktionen auf der Arbeitsfläche:
- **Auf einen Node klicken** öffnet dessen Einstellungspanel auf der rechten Seite
- **Vom Output eines Nodes ziehen** (der Punkt auf der rechten Seite) erstellt eine Verbindung zum nächsten Node
- **Doppelklick auf leeren Bereich** öffnet die Node-Suche
- **Strg/Cmd + Klick** wählt mehrere Nodes aus
- **Strg/Cmd + C / V** kopiert und fügt Nodes zwischen Workflows ein

### Das Node-Panel

Klicken Sie auf den **+**-Button oder doppelklicken Sie auf die Arbeitsfläche, um die Node-Suche zu öffnen. Sie können nach Kategorie (Trigger, Action, Flow usw.) browsen oder nach Name suchen. Kategorien, die Sie am häufigsten nutzen:

- **Trigger-Nodes:** Starten einen Workflow (Webhook, Schedule, App-Trigger, Manual)
- **Action-Nodes:** Führen etwas aus (E-Mail senden, CRM-Kontakt erstellen, API aufrufen)
- **Flow-Nodes:** Steuern die Logik (If, Switch, Merge, Split, Wait, Loop)
- **Code-Nodes:** JavaScript oder Python für individuelle Transformationen ausführen
- **KI-Nodes:** Verbindung zu OpenAI, Anthropic oder lokalen LLMs

### Das Ausführungs-Panel

Nach dem Ausführen eines Workflows (per Test oder Produktions-Trigger) zeigt das Ausführungs-Panel die Daten, die durch jeden Node fließen. Klicken Sie auf einen beliebigen Node, um dessen Ein- und Ausgabedaten zu sehen. Dies ist das primäre Debugging-Tool -- wenn ein Node unerwartete Daten empfängt, sehen Sie es hier sofort.

### Steuerleiste oben

- **Test workflow:** Führt den Workflow einmal mit dem aktuellen Trigger aus (oder mit gepinnten Testdaten)
- **Active-Schalter:** Schaltet den Workflow ein (Produktionsmodus) oder aus
- **Ausführungshistorie:** Zeigt vergangene Durchläufe mit vollständigen Ein-/Ausgabedaten und Fehlerprotokollen
- **Einstellungen (Zahnrad-Icon):** Workflow-weite Einstellungen wie Fehlerbehandlung, Zeitzone und Ausführungs-Timeout konfigurieren

## Ersten Workflow bauen: Ein Mini-Workflow

Hier ist der einfachste mögliche Workflow, um die Kernkonzepte zu lernen. Er dauert unter 5 Minuten.

### Ziel: Webhook empfängt Daten, sendet Benachrichtigung

Dieser Mini-Workflow hat zwei Nodes: einen Webhook-Trigger und eine Slack-Benachrichtigung. Wenn Daten an die Webhook-URL gesendet werden, wird eine Slack-Nachricht verschickt.

**Schritt 1:** Erstellen Sie einen neuen Workflow (klicken Sie oben links auf "Add workflow").

**Schritt 2:** Die Arbeitsfläche zeigt standardmäßig einen Manual Trigger. Löschen Sie ihn und fügen Sie einen **Webhook**-Node hinzu. n8n generiert eine eindeutige URL. Kopieren Sie diese -- Sie brauchen sie zum Testen.

**Schritt 3:** Fügen Sie einen **Slack**-Node hinzu (oder Gmail, Microsoft Teams oder einen anderen Benachrichtigungsdienst). Verbinden Sie ihn mit dem Webhook-Node, indem Sie vom Output-Punkt des Webhooks zum Input-Punkt des Slack-Nodes ziehen.

**Schritt 4:** Konfigurieren Sie den Slack-Node. Wählen Sie Ihren Slack-Workspace aus (einmalige OAuth-Verbindung), wählen Sie einen Kanal und schreiben Sie die Nachricht. Nutzen Sie n8n-Expressions, um dynamische Daten einzufügen: `{{ $json.name }}` fügt das "name"-Feld aus dem Webhook-Payload ein.

**Schritt 5:** Klicken Sie auf "Test workflow." n8n wechselt in den Listening-Modus. Senden Sie einen Test-Request an die Webhook-URL mit curl:

```bash
curl -X POST https://your-n8n-instance/webhook/xxx \
  -H "Content-Type: application/json" \
  -d '{"name": "Lisa", "company": "Müller GmbH"}'
```

Wenn beide Nodes grün aufleuchten, funktioniert es. Prüfen Sie Slack auf die Nachricht.

**Schritt 6:** Schalten Sie den **Active**-Schalter um. Der Workflow ist jetzt live.

Sie verstehen nun Trigger, Nodes, Verbindungen, Expressions, Tests und Aktivierung. Jeder n8n-Workflow, egal wie komplex, wird aus diesen gleichen Bausteinen aufgebaut.

## Einen realen Workflow bauen: Formular zu Tabelle zu CRM

Jetzt bauen wir etwas Praktisches. Dieser Workflow erfasst eine Webformular-Eingabe, protokolliert sie in Google Sheets, erstellt einen Kontakt im CRM und sendet eine Benachrichtigung.

### Schritt 1: Webhook-Trigger

Fügen Sie wie zuvor einen Webhook-Node hinzu. Das ist der Einstiegspunkt für Ihr Webformular. Konfigurieren Sie Ihr Formular so, dass es Daten per POST an die Webhook-URL sendet.

### Schritt 2: In Google Sheets protokollieren

Klicken Sie auf **+** und fügen Sie einen **Google Sheets**-Node hinzu. Konfiguration:
1. Google-Konto verbinden (einmalige Autorisierung)
2. **"Append Row"** als Aktion wählen
3. Ziel-Spreadsheet und Sheet auswählen
4. Eingehende Webhook-Daten auf Spalten mappen

Die visuelle Mapping-Oberfläche zeigt exakt, welche Datenfelder vom Trigger zum Google-Sheets-Node fließen.

### Schritt 3: CRM-Kontakt erstellen

Fügen Sie einen **Customermates**-Node hinzu (oder HubSpot, Pipedrive oder Ihr CRM). Konfiguration:
1. CRM-Account verbinden
2. **"Create Contact"** als Aktion wählen
3. Name, E-Mail, Firma und andere Felder aus den Webhook-Daten mappen

Bei [Customermates](/) (10 EUR/Nutzer/Monat) ist die n8n-Integration nativ -- dedizierte Nodes statt roher API-Aufrufe.

### Schritt 4: Benachrichtigung senden

Fügen Sie einen **Slack**-Node hinzu, um das Team zu informieren, dass ein neuer Lead eingegangen ist. Fügen Sie den Kontaktnamen, die Firma und einen Link zum CRM-Datensatz ein.

### Schritt 5: Testen und aktivieren

Testen Sie die gesamte Kette mit Beispieldaten. Prüfen Sie, ob die Zeile in Google Sheets erscheint, der Kontakt im CRM existiert und die Slack-Nachricht angekommen ist. Dann aktivieren.

## Workflows designen: Best Practices

Bevor Sie komplexe Automatisierungen bauen, lernen Sie Prinzipien, die spätere Probleme vermeiden.

### Workflows fokussiert halten

Jeder Workflow sollte eine Sache gut machen. Ein Workflow, der Leads erfasst, bewertet, CRM-Einträge erstellt, E-Mails sendet und ein Dashboard aktualisiert, ist schwer zu debuggen, wenn etwas bricht. Teilen Sie stattdessen in fokussierte Sub-Workflows auf:

1. **Lead-Erfassungs-Workflow:** Webhook zu CRM
2. **Lead-Scoring-Workflow:** CRM-Trigger zu ChatGPT zu CRM-Update
3. **Benachrichtigungs-Workflow:** CRM-Trigger zu Slack

### Namenskonventionen verwenden

Benennen Sie Ihre Workflows und Nodes aussagekräftig. "Workflow 1" sagt nichts aus. "Lead-Erfassung: Website-Formular zu Customermates" sagt alles. Benennen Sie Nodes nach ihrer Funktion: "Kontakt in CRM erstellen" statt "HTTP Request 3."

### Testdaten pinnen

Nutzen Sie beim Entwickeln die "Pin data"-Funktion, um Beispiel-Eingaben für einen Node zu speichern. So können Sie nachgelagerte Nodes testen, ohne jedes Mal den gesamten Workflow neu auszulösen.

### Die Workflow-Bibliothek nutzen

Die [n8n-Workflow-Bibliothek](https://n8n.io/workflows/) enthält über 1.000 fertige Templates. Durchsuchen Sie nach Kategorie (CRM, Marketing, Vertrieb, DevOps) und kopieren Sie ein Template in Ihren Workspace. Ein bestehender Workflow lässt sich schneller anpassen als einer von Grund auf neu gebaut werden.

## Credentials-Verwaltung

Jeder externe Dienst, den Sie mit n8n verbinden, erfordert Anmeldedaten (Credentials). Die richtige Verwaltung ist wichtig für Sicherheit und Team-Zusammenarbeit.

### Wie Credentials in n8n funktionieren

Credentials werden getrennt von Workflows gespeichert. Wenn Sie zum ersten Mal Google Sheets verbinden, speichert n8n das OAuth-Token als Credential. Sie können dieses Credential dann in mehreren Workflows wiederverwenden, ohne sich erneut zu authentifizieren.

### Best Practices für Credentials

- **Benennen Sie Credentials klar.** Verwenden Sie "Google Sheets - Vertriebsteam" statt "Google Sheets 1." Wenn ein Teammitglied einen Workflow übernimmt, muss es wissen, welches Konto verbunden ist.
- **Nutzen Sie Service-Accounts, wenn möglich.** Für Google Workspace und andere Enterprise-Tools verwenden Sie einen dedizierten Service-Account statt eines persönlichen Kontos. Das verhindert, dass Workflows brechen, wenn jemand das Team verlässt.
- **Rotieren Sie API-Keys regelmäßig.** Setzen Sie eine Kalendererinnerung, um Keys alle 90 Tage für sicherheitsrelevante Integrationen zu rotieren.
- **Beschränken Sie den Zugriff bei Self-Hosting.** Bei selbst gehostetem n8n verwenden Sie Umgebungsvariablen für sensible Credentials, statt sie in der Oberfläche einzugeben. So bleiben Geheimnisse aus Workflow-Exporten heraus.
- **Testen Sie nach Credential-Änderungen.** Nach dem Aktualisieren eines Credentials (neuer API-Key, erneute OAuth-Authentifizierung) testen Sie jeden Workflow, der es verwendet.

### Häufige Credential-Probleme

| Problem | Ursache | Lösung |
|---------|---------|--------|
| "Invalid credentials"-Fehler | Token abgelaufen oder widerrufen | In Credential-Einstellungen erneut authentifizieren |
| Workflow lief gestern, scheitert heute | OAuth-Token-Refresh fehlgeschlagen | Credential löschen und neu erstellen |
| Verbindung zu Self-Hosted-Dienst nicht möglich | Firewall- oder Netzwerkproblem | Sicherstellen, dass n8n die Service-URL erreichen kann |
| Credential im Node nicht verfügbar | Falscher Credential-Typ ausgewählt | Prüfen, ob das Credential zum Node passt |

## Fehlerbehandlung: Resiliente Workflows bauen

Workflows scheitern. APIs haben Timeouts, Anmeldedaten laufen ab, Datenformate ändern sich, Rate-Limits greifen. Wenn Sie Fehlerbehandlung von Anfang an einbauen, sparen Sie später Stunden beim Debugging.

### Der Error-Trigger-Workflow

Erstellen Sie einen dedizierten Fehlerbehandlungs-Workflow, der Ausfälle über alle Ihre Workflows hinweg abfängt:

1. Fügen Sie einen **Error Trigger**-Node hinzu (unter Trigger-Nodes). Dieser wird ausgelöst, wann immer ein Workflow fehlschlägt.
2. Verbinden Sie ihn mit einem **Slack**- oder **E-Mail**-Node, der die Fehlerdetails sendet (Workflow-Name, Node-Name, Fehlermeldung, Zeitstempel).
3. Optional: Fügen Sie einen **Google Sheets**-Node hinzu, um Fehler für spätere Analyse zu protokollieren.

Dieser einzelne Workflow gibt Ihnen Monitoring über Ihre gesamte n8n-Instanz.

### Fehlerbehandlung auf Node-Ebene

Für kritische Nodes konfigurieren Sie das Fehlerverhalten:

- **Retry on Failure:** Setzen Sie einen Node so, dass er bei Fehler 2-3 Mal mit Verzögerung erneut versucht. Das fängt kurzfristige API-Timeouts ab.
- **Continue on Error:** Wenn ein unkritischer Node fehlschlägt (z. B. Protokollierung in einer Tabelle), läuft der Workflow weiter. Der Fehler wird protokolliert, blockiert aber nicht den Hauptprozess.
- **Error-Output:** Einige Nodes haben einen Fehler-Output-Zweig. Verbinden Sie ihn mit einem alternativen Pfad (z. B. an eine Dead-Letter-Queue senden oder einen Admin benachrichtigen).

### Datenvalidierung mit If-Nodes

Nutzen Sie **If**-Nodes, um Grenzfälle zu behandeln, bevor sie Fehler verursachen:

- Prüfen Sie, ob ein Pflichtfeld existiert, bevor es an den nächsten Node weitergegeben wird
- Validieren Sie das E-Mail-Format, bevor es an ein CRM gesendet wird
- Prüfen Sie, ob eine Liste leer ist, bevor Elemente verarbeitet werden
- Verifizieren Sie, dass eine API-Antwort die erwarteten Felder enthält

### Timeout-Konfiguration

Für Workflows, die langsame externe APIs aufrufen, erhöhen Sie das Node-Timeout (Standard: 300 Sekunden). Für Workflows, die per Webhook ausgelöst werden und schnell antworten müssen, setzen Sie ein kurzes Timeout und verarbeiten die Hauptlogik asynchron.

## Community Nodes: n8n über 400 Integrationen hinaus erweitern

Die über 400 integrierten Nodes von n8n decken die meisten populären Dienste ab. Für alles andere gibt es Community Nodes.

### Was sind Community Nodes?

Community Nodes sind Integrationen, die von der n8n-Community erstellt und auf npm veröffentlicht werden. Sie folgen derselben Node-Struktur wie integrierte Nodes und erscheinen nach der Installation im Node-Panel.

### Community Nodes installieren

**Auf n8n Cloud:** Gehen Sie zu Einstellungen > Community Nodes, suchen Sie den Node und installieren Sie ihn mit einem Klick.

**Bei Self-Hosting:** Fügen Sie das Node-Paket zu Ihrer n8n-Umgebung hinzu:

```bash
# In Ihrem n8n-Docker-Container oder der npm-Installation:
npm install n8n-nodes-your-community-node
```

Starten Sie dann n8n neu. Der neue Node erscheint im Node-Panel.

### Beliebte Community Nodes

- **n8n-nodes-text-manipulation:** Erweiterte String-Operationen über die eingebauten Expressions hinaus
- **n8n-nodes-document-generator:** PDFs und Word-Dokumente aus Templates erstellen
- **n8n-nodes-puppeteer:** Web-Scraping und Browser-Automatisierung
- **Diverse CRM-spezifische Nodes:** Integrationen für Nischen-CRMs, die nicht durch eingebaute Nodes abgedeckt sind

### Vorsicht bei Community Nodes

Community Nodes werden von ihren Autoren gepflegt, nicht von n8n. Bevor Sie einen in Produktion nutzen:

- Prüfen Sie das letzte Update-Datum (vermeiden Sie Nodes, die seit 6+ Monaten nicht aktualisiert wurden)
- Lesen Sie die GitHub-Issues auf bekannte Bugs
- Testen Sie gründlich in einer Entwicklungsumgebung
- Haben Sie einen Fallback-Plan (z. B. HTTP-Request-Node als Alternative)

## CRM-Integrations-Beispiele mit n8n

Die Verbindung von n8n mit einem CRM erschließt leistungsfähige Automatisierungen. Hier sind vier praktische Beispiele mit [Customermates](/) als CRM (dieselben Muster funktionieren mit HubSpot, Pipedrive und anderen).

### Beispiel 1: Leads aus Webformularen automatisch importieren

```
Webformular-Eingabe -> Webhook -> Customermates (Kontakt erstellen)
```

Jede Formular-Eingabe auf Ihrer Website erstellt automatisch einen neuen Kontakt in Ihrem CRM, wobei die übermittelten Felder auf Kontakteigenschaften gemappt werden. Keine manuelle Dateneingabe.

### Beispiel 2: KI-gestützte Lead-Qualifizierung

```
Neuer Kontakt im CRM -> OpenAI-Node (Lead bewerten) -> Customermates (Kontakt mit Score aktualisieren) -> Slack (Benachrichtigung bei Hot Lead)
```

Wenn ein neuer Lead ins CRM gelangt, sendet n8n die Lead-Daten an ChatGPT für BANT-Scoring, schreibt den Score in ein Custom-Feld zurück und benachrichtigt das Vertriebsteam auf Slack, wenn der Lead als "hot" klassifiziert wird. Die vollständige Einrichtung finden Sie in unserem Leitfaden zur [Lead-Qualifizierung mit ChatGPT](/blog/lead-qualification-chatgpt).

### Beispiel 3: Automatisierte Follow-up-E-Mails

```
Schedule Trigger (täglich) -> Customermates (Deals ohne Aktivität seit 7 Tagen) -> Gmail (Follow-up senden)
```

Jeden Morgen prüft n8n, welche Deals seit 7 Tagen keine Aktivität hatten, und versendet automatisch eine Follow-up-E-Mail an den Kontakt. Der E-Mail-Inhalt kann mit Deal-Daten aus dem CRM personalisiert werden.

### Beispiel 4: Gesprächsnotizen ins CRM

```
Meeting-Tool-Webhook -> OpenAI-Node (Zusammenfassen) -> Customermates (Notiz am Kontakt erstellen)
```

Nach jedem Kundengespräch wird das Meeting-Transkript automatisch von ChatGPT zusammengefasst und als Notiz am relevanten Kontakt im CRM hinterlegt. Die vollständige Einrichtung finden Sie in unserem Leitfaden zum [Zusammenfassen von Gesprächsnotizen mit ChatGPT](/blog/chatgpt-summarize-notes).

### Warum Customermates gut mit n8n zusammenarbeitet

Customermates (10 EUR/Nutzer/Monat, alle Funktionen inklusive) bietet eine native n8n-Integration. Das bedeutet, dass Sie in n8n dedizierte Customermates-Nodes verwenden können, anstatt rohe API-Aufrufe zu konfigurieren. Die Integration unterstützt das Erstellen, Lesen, Aktualisieren und Löschen von Kontakten, Deals, Organisationen und Notizen.

Für Teams, die n8n als Automatisierungs-Backbone einsetzen, beseitigt ein CRM mit nativer n8n-Unterstützung die Reibungsverluste manueller API-Konfiguration und reduziert die Einrichtungszeit von Stunden auf Minuten.

## Einen mehrteiligen Workflow bauen: Datenquelle zu CRM

Sobald Sie mit grundlegenden Workflows vertraut sind, ist es Zeit für ein komplexeres Beispiel. Dieser Workflow spiegelt einen realen Anwendungsfall wider: Daten aus einer Quelle ziehen, filtern und transformieren, Berechnungen durchführen und Ihr CRM aktualisieren -- mit Benachrichtigungen und Zeitplanung.

### Das Szenario

Jeden Montagmorgen soll Ihr Workflow:
1. Die Vertriebsdaten der letzten Woche aus einer Airtable-Basis (oder Datenbank) ziehen
2. Nach Deals über einem Schwellenwert filtern
3. Einen Leistungswert für jeden Vertriebsmitarbeiter berechnen
4. Die entsprechenden Kontakte im CRM aktualisieren
5. Eine Zusammenfassung an das Team senden

### Schritt-für-Schritt-Aufbau

**1. Schedule Trigger:** Eingestellt auf "Jeden Montag um 8:00 Uhr." Dies ersetzt den Webhook -- der Workflow läuft nach Zeitplan statt on-demand.

**2. Airtable-Node (oder Datenbank-Node):** Verbinden Sie sich mit Ihrer Airtable-Basis und ziehen Sie Datensätze aus der "Deals"-Tabelle für die letzten 7 Tage. Bei PostgreSQL oder MySQL verwenden Sie den entsprechenden Datenbank-Node mit einer datumsgefilterten Abfrage.

**3. If-Node (Filter):** Filtern Sie die Ergebnisse, um nur Deals über 1.000 EUR einzuschließen. Das konzentriert die nachgelagerte Verarbeitung auf signifikante Deals.

**4. Code-Node (Berechnungen):** Verwenden Sie einen JavaScript-Code-Node, um Kennzahlen zu berechnen:

```javascript
// Deals nach Vertriebsmitarbeiter gruppieren und Summen berechnen
const grouped = {};
for (const item of $input.all()) {
  const rep = item.json.salesRep;
  if (!grouped[rep]) {
    grouped[rep] = { rep, totalValue: 0, dealCount: 0 };
  }
  grouped[rep].totalValue += item.json.dealValue;
  grouped[rep].dealCount += 1;
}
return Object.values(grouped).map(g => ({ json: g }));
```

**5. CRM-Node (Update):** Für jeden Vertriebsmitarbeiter aktualisieren Sie den Kontakt-Datensatz in Customermates mit den neuesten Leistungskennzahlen.

**6. Slack-Node (Benachrichtigung):** Senden Sie eine formatierte Zusammenfassung an einen Slack-Kanal:

```
Wöchentliche Vertriebsübersicht:
{{ $json.rep }}: {{ $json.dealCount }} Deals, {{ $json.totalValue }} EUR
```

**7. Aktivieren.** Der Workflow läuft jeden Montag automatisch. Wenn Sie ihn sofort testen möchten, nutzen Sie den "Test workflow"-Button -- er führt die gesamte Kette unabhängig vom Zeitplan aus.

### Workflows exportieren und importieren

n8n-Workflows werden als JSON gespeichert. Sie können sie frei exportieren und importieren:

**Exportieren:** Klicken Sie auf das Drei-Punkte-Menü bei einem Workflow und wählen Sie "Download." Die JSON-Datei enthält jeden Node, jede Verbindung und Konfiguration (aber keine Credentials).

**Importieren:** Klicken Sie beim Erstellen eines neuen Workflows auf "Import from file." Wählen Sie die JSON-Datei. Aktualisieren Sie die Credentials passend zu Ihrer Umgebung, und der Workflow ist bereit.

Das erleichtert:
- Workflows mit Teammitgliedern teilen
- Workflows vor Änderungen sichern
- Workflows zwischen n8n-Instanzen verschieben (z. B. Entwicklung zu Produktion)
- Community-Templates aus der [n8n-Workflow-Bibliothek](https://n8n.io/workflows/) nutzen

## Vom Anfänger zum Fortgeschrittenen: Nächste Schritte

Sobald Sie mit grundlegenden Workflows vertraut sind, bietet n8n erweiterte Möglichkeiten:

- **Sub-Workflows:** Einen Workflow aus einem anderen aufrufen, um modulare, wiederverwendbare Automatisierungsbausteine zu schaffen.
- **Code-Nodes:** JavaScript oder Python für Logik schreiben, die sich mit Standard-Nodes nicht abbilden lässt.
- **KI-Agent-Nodes:** Mehrstufige native KI-Integration bauen, die Tools nutzen, Entscheidungen treffen und autonom mit Ihrem CRM interagieren.
- **Datenbank-Nodes:** Direkte Verbindung zu PostgreSQL, MySQL, MongoDB und anderen Datenbanken.

## Häufig gestellte Fragen

### Ist n8n wirklich kostenlos?

Selbst gehostetes n8n ist kostenlos nutzbar mit unbegrenzten Workflow-Ausführungen. Der Quellcode ist offen und auf GitHub verfügbar. n8n Cloud ist ein kostenpflichtiger Managed Service ab 20 EUR/Monat. Die meisten Einsteiger beginnen mit dem Cloud-Plan und wechseln zum Self-Hosting, wenn die Nutzung wächst.

### Brauche ich Programmierkenntnisse, um n8n zu nutzen?

Nein. Der visuelle Workflow-Editor deckt die meisten Automatisierungs-Anwendungsfälle ohne Code ab. Für fortgeschrittene Transformationen bietet n8n Code-Nodes, in denen Sie JavaScript oder Python schreiben können, diese sind aber optional. Wenn Sie Ihre Automatisierung als "Wenn X passiert, mache Y" beschreiben können, können Sie sie in n8n bauen.

### Wie schneidet n8n im Vergleich zu Zapier bei CRM-Automatisierung ab?

n8n ist flexibler und bei wachsendem Volumen deutlich günstiger. Zapier punktet mit der breitesten Auswahl an vorgefertigten Integrationen (6.000+ vs. 400+), aber n8n bietet unbegrenzte Ausführungen beim Self-Hosting, native KI/LLM-Unterstützung, Custom Code und volle Datenkontrolle. Speziell für CRM-Automatisierung machen die tieferen Integrationsmöglichkeiten und KI-Nodes von n8n es zur stärkeren Plattform.

### Kann n8n sich mit jedem CRM verbinden?

n8n hat dedizierte Nodes für verbreitete CRMs, darunter Customermates, HubSpot, Pipedrive, Salesforce und Zoho. Für CRMs ohne dedizierten Node ermöglicht der HTTP-Request-Node die Anbindung an jedes CRM mit REST-API. Community Nodes erweitern die Möglichkeiten noch weiter. In der Praxis deckt das praktisch jedes moderne CRM-System ab.

### Ist n8n für den Produktiveinsatz geeignet?

Ja. Tausende Unternehmen betreiben n8n in Produktion. Für Zuverlässigkeit hosten Sie auf einem VPS mit Docker, konfigurieren persistenten Speicher und automatisierte Backups und richten Monitoring ein. Die n8n-Cloud-Option übernimmt all das für Sie. Für geschäftskritische Workflows fügen Sie Fehlerbehandlung und Benachrichtigungen hinzu, damit Sie sofort erfahren, wenn etwas fehlschlägt.

### Wie geht n8n mit DSGVO und Datenschutz um?

Selbst gehostetes n8n gibt Ihnen die vollständige Kontrolle -- Daten verlassen nie Ihre Server. Bei n8n Cloud werden Daten in der EU verarbeitet. In beiden Fällen greift n8n selbst nicht auf Ihre Workflow-Daten zu und speichert sie nicht über die Ausführung hinaus. In Kombination mit einem DSGVO-konformen CRM wie Customermates (EU-gehostet, 10 EUR/Nutzer/Monat) lassen sich vollständig konforme Automatisierungs-Pipelines aufbauen.

### Was ist der einfachste erste Workflow?

Beginnen Sie mit einem Webhook-zu-Benachrichtigung-Workflow: Daten per Webhook empfangen, Slack- oder E-Mail-Benachrichtigung senden. Das vermittelt die Kernkonzepte (Trigger, Node, Datenfluss, Test, Aktivierung) in unter 10 Minuten. Sobald das läuft, fügen Sie einen CRM-Node zum Speichern der Daten hinzu, und Sie haben eine praktische Lead-Erfassungs-Automatisierung.

### Wie behandle ich Fehler in n8n-Workflows?

Erstellen Sie einen globalen Error-Trigger-Workflow, der Sie benachrichtigt, wenn ein Workflow fehlschlägt. Auf Node-Ebene nutzen Sie Retry-on-Failure für API-Aufrufe, Continue-on-Error für unkritische Schritte und If-Nodes für Datenvalidierung. Die Ausführungshistorie in n8n protokolliert jeden Durchlauf mit vollständigen Ein-/Ausgabedaten und macht Debugging unkompliziert.

### Kann ich Community Nodes auf n8n Cloud nutzen?

Ja. n8n Cloud unterstützt die Installation von Community Nodes über das Einstellungsmenü. Bei selbst gehosteten Instanzen installieren Sie diese via npm. Testen Sie Community Nodes immer in einer Entwicklungsumgebung, bevor Sie sie in Produktions-Workflows einsetzen.
